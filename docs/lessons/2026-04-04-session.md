### 1. Tarjima tizimi (next-intl) — asosiy muammo hal qilindi

**Muammo:**  
Til almashtirish tugmasi (NL/EN) URL ni to'g'ri o'zgartirardi, lekin sahifa matnlari o'zgarmasdi. Barcha matnlar har doim Dutch tilida qolardi.

**Sabab:**  
`setRequestLocale(locale)` funksiyasi hech bir joyda chaqirilmagan edi. next-intl v4 + `generateStaticParams` kombinatsiyasida bu **majburiy**. Agar chaqirilmasa, `getMessages()` qaysi locale ishlatishni bilmaydi va har doim default (`nl`) ishlatadi.

**Yechim:**
```typescript
// app/[locale]/layout.tsx va barcha page.tsx larga qo'shildi
import { setRequestLocale } from "next-intl/server";

export default async function Page({ params }) {
  const { locale } = await params;
  setRequestLocale(locale); // ← Bu chaqiruv majburiy!
  ...
}
```

**Qo'shilgan fayllar:** layout.tsx + 9 ta page fayl

**Qo'shimcha:** `NextIntlClientProvider` ga `locale={locale}` prop ham qo'shildi — client komponentlar (Hero, Header va h.k.) uchun zarur.

### 5. Hydration Mismatch xatosi — `not-found.tsx`

**Muammo:**
```
A tree hydrated but some attributes of the server rendered HTML didn't match
at body — RootNotFound (src/app/not-found.tsx:10:7)
```

**Sabab:**  
`app/not-found.tsx` ichida `<html>` va `<body>` teglari yaratilgan edi. Lekin Next.js App Router bu komponentni `app/layout.tsx` ichida render qiladi — ya'ni `<html>` ichida yana `<html>` bo'lib, React hydration crash berdi. `ThemeProvider` ham `<html>` ga `class="dark"` qo'shganidan conflict chiqdi.

**Yechim:**
```tsx
// Xato (nested html):
export default function RootNotFound() {
  return (
    <html lang="nl">  // ← BU XATO!
      <body>...</body>
    </html>
  );
}

// To'g'ri (faqat kontent):
export default function RootNotFound() {
  return (
    <div style={{ display: "flex", minHeight: "70dvh", ... }}>
      <h1>404</h1>
      <Link href="/nl">Terug naar home</Link>
    </div>
  );
}
```

## Muhim darslar (Lessons Learned)

### next-intl v4 qoidalari
1. `setRequestLocale(locale)` — **layout + har bir page** da chaqirilishi shart
2. `generateStaticParams` bilan birga ishlatilganda bu yanada muhimroq
3. `NextIntlClientProvider` ga `locale={locale}` prop berishni unutma
4. `getLocale()` — param yo'q joylarda (not-found, layout) locale olish uchun

### Next.js App Router qoidalari
1. `not-found.tsx` — App Router uni layout ichida render qiladi, `<html>/<body>` YO'Q
2. `error.tsx` — majburiy "use client", `reset()` funksiyasi oladi
3. Server komponentlar va Client komponentlar orasida state aralashib ketmasin

### Dark Mode
1. CSS variables yondashuvi eng oson — `.dark` selektor bilan override
2. `next-themes` — `suppressHydrationWarning` root `<html>` da bo'lishi shart
3. ThemeToggle `mounted` state — SSR/CSR mismatch oldini oladi

### Tailwind CSS v4
1. `@plugin "@tailwindcss/typography"` — v3 da `@import` edi
2. Dark mode uchun `.dark` class selektor — media query emas
3. CSS variables `var()` bilan — tema o'zgarishida hamma joy bir zumda yangilanadi
